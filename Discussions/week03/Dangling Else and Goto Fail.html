Good idea! The "goto fail" security bug that made the news a few years ago was caused by not using curly braces around an if-statement body. 

As I recall, the code looked something like this (it was written in C):

if (some_condition == some_other_condition)
    do_something_important();
    goto fail;

The problem is that the indentation makes the reader assume those statements will be executed if the condition is true. But only the first line will be executed. The "goto fail" line will be executed every time because it's not part of an if-statement block. It only appears to be.

So, as a rule, I use curly braces around even single-line blocks of code. It's just smart and safe.
