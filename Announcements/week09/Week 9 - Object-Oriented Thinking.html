Abstraction

Abstraction is nothing more than describing the data and actions associated with the real-world things we need to model in our programs to solve some problem.

For example, if we expand on our Family class from last week, the family is made of up of individual people. A person has a name, a gender, and a date of birth, from which we can calculate their age. Check out the UML for these relationships:

Abstraction is the process of breaking down our real world objects into describable parts that we can code in software. As you can see between last week and this week, we can further refine our abstraction until it solves the problems we're needing to solve with our software. Notice that there are still aspects of a family that we don't model, such as which child is the favorite. (Just kidding!)

Our abstractions doesn't have to try to model every detail we can ever think of. That would be like trying to boil the ocean with a hot plate. We only need to describe the things that help us solve the problems our software needs to solve.

Encapsulation

Encapsulation is the principle of not worrying about how things are implemented. This is from the perspective of the user of the class. The author of the class cares very much about how it is implemented. But the author can tell the user that the way the person's age is calculated is none of their business, so long as the answer provided by the getAge() method is correct.

Take a look at the code for our Family model in the Week 9 Code Samples.

"Contract"

Abstraction and Encapsulation combine to create a "contract" between the implementer and the user of a class. The contract (or interface) of the class describes how the class works. This is where the documentation for the class methods becomes important.

Good old Pythagorus (he had this theorem)

You'll need to use the Pythagorean theorem this week in order to calculate the distance between two points. The theorem is, for right triangles, the sides are related to each other like this: a2 + b2 = c2. 

The distance between two points on a graph is the hypotenuse, or "c", of a right triangle. So if we can find the length of the other two sides of the triangle (the "a" and "b"), we can easily find the "c" like this:

double c = Math.sqrt( (a*a) + (b*b) );

You can find "a" and "b" by subtracting the x and y coordinates from each point:

double a = point1.x - point2.x;
double b = point1.y - point2.y;

The D.R.Y. Principle

There is a principle in software development known as the DRY principle. It is an acronym that stands for Don't Repeat Yourself. 

Our assignment this week will include writing two methods that both return the distance between two points. You only want to write that Pythagorean stuff once. You'll need to figure out how to reuse it for the second method.
