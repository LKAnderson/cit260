Objects and Classes

Up until now, we have been using very basic data and solving some simple (relatively speaking) problems. Computer software is supposed to help us with the complex stuff. Objects and classes will help us do that in a way that helps us maintain our sanity!

Imagine that you need to write a program that manages families. The program must store the Surname, Father's name, Mother's name, and the names of each of the children. With what we have learned up until now, how might we do this? Check out this solution with arrays:

package officehours;
    
    public class FamilyArray {
    
        public static void main(String[] args) {
            
            
            String[][] families = new String[][] {
                
                { "Smith", "Joseph", "Lucy", "Alvin", "Hyrum", 
                    "Sophronia", "Jospeh", "Samuel Harrison", 
                    "Ephraim", "William", "Katharine", 
                    "Don Carlos", "Lucy" },    
            };
            
        }
    }

We could say the surname is element 0, the father's name is element 1, the mother's name is element 2, and everything else is the kids' names. That'd be tough to remember. We could be a little smarter about it:

package officehours;
    
    public class FamilyArray {
    
        public static void main(String[] args) {
            
            final int SURNAME = 0;
            final int FATHER = 1;
            final int MOTHER = 2;
            final int KIDS_START = 3;
            
            String[][] families = new String[][] {
                
                { "Smith", "Joseph", "Lucy", "Alvin", "Hyrum", 
                    "Sophronia", "Jospeh", "Samuel Harrison", 
                    "Ephraim", "William", "Katharine", 
                    "Don Carlos", "Lucy" },    
            };
            
        }
    }

But even that is going to be really hard to deal with.

Classes give us a way to define more complex data structures, building them up using the primitive types we have been using. It is also possible to include complex classes in other classes. This flexibility is what makes Object Oriented Programming so powerful. We can model pretty much everything we can describe.

UML Diagramming

One of the programming assignments this week asks you to create a UML diagram (Unified Modeling Language). Don't get hung up on this. There are plenty of examples of it in the book, and you only have to model one simple class with it. For the assignment, you can draw it by hand and submit a photo of it. Or you can use drawing software that supports UML. I like a free tool called Draw.io (https://www.draw.io). In the help menu on that site, there is a link to download the desktop version.
