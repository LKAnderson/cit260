How to Read Chapter 6

There is a lot of technical talk in Chapter 6. 

It all boils down to this: Programmers hate repeating themselves. We love writing reusable code and calling over and over. Methods are nothing more than a way to pull that reusable code out of our main() method so we can call it over and over.

Some methods can do everything they need without any extra information. 

Some methods want us to provide some input to them so they can make decisions based on it.

And some methods provide a general algorithm, but need us to provide some of the values that would be different each time it is used. These are "variables." That term should not be new to you :-) We pass these in as inputs, too.

The inputs to a method are called its "parameters." The parameters become local variables within the scope of the method's code. When the method is done, its parameters "evaporate."

If we need a method to return a value, we have to return the value using the "return" statement. "Function" is what we call a method that returns a value. (The book also calls it a "value returning method." It's easier to say "function")

Parameters are passed by value, meaning a copy of the value is given to the method to use. Nothing happens to the value outside of the method. (We will learn more about this in future weeks. Classes and Objects will introduce pass-by-reference, but it will make so much more sense after you've taken the time to understand passing by value.)

There is some talk about activation records and an area of memory called a stack. This is nothing more than a way to provide the copied values to the method, and for the Java runtime to know which instruction to return to after the method ends.
