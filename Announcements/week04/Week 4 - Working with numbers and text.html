This week's lesson is about working with the methods found in the Math class. DON'T PANIC: This is not a math class. You don't have to memorize any formulas. Just become familiar with the functions that are available in the Math class.

Our chapter also introduces the basics of working with text in our programs. We'll be getting familiar with the char data type, and the Character and String classes.

We know enough to be creative

With this week's topics, we now know enough to afford us some creativity in the way we solve the problems in the programming exercises. I'm looking forward see how you solve the problems. These exercises will require quite a bit of thought in order to solve them.

From this point on, all of your programs need to check for invalid user input. I'll be throwing some invalid data at your programs to see how they handle it. 

Assignment Rubrics

Our pilot course does not provide a rubric for the programming exercises. I have been using my own rubric to make sure I am consistent and fair in my grading. I've added official rubrics to the remainder of the lessons in the course. 

For each programming exercise, I evaluate your work on a scale from 0 to 10 in the following three areas.

  • (40%) Correct implementation of the requirement. The description of the programming exercise is our requirement. I am looking for how closely your code matches the requirement(s) given for the exercise.
  • (55%) Demonstrated mastery of the week's subject matter. I want to see that you understand what we've been learning each week.
  • (5%) Consistent and appropriate code formatting. 

Finally, it is expected that you will compile and test your program before submitting it. If you submit code that does not compile or run, you will lose 50% of your score. I do give you partial credit for the work you put in, but if your code does not compile or run, it is not acceptable as a submission.
